A short demo of using actors in dynamically loaded views

This project uses two AngularActors: OuterPingActor and InnerPingActor. These actors simply send pings to the client, which are then picked up and logged to the JS console. You can use your browsers dev tools to view them.

OuterPing is loaded in the "outer" context of the application, i.e. a page served directly by Lift, not loaded dynamically by Angular. The converse is true for InnerPing, which is loaded on a page which is retrived and served by Angular, by means of the ng-view attribute.

You can use the buttons below to start and stop the ping process for the outer context. Check your console to verify that the pings indeed arrive.

When ready, follow the "Angular dynamic view" in the menu, in order to dynamically load an Angular view with similar functionality.
